## Why This Manifesto?

Agentic AI systems can now:

- Read repositories  
- Modify files  
- Execute commands  
- Generate large portions of production code  

Their productivity impact is undeniable.

But if adopted without governance, they risk eroding:

- Human accountability  
- Design competence  
- Deep validation practices  
- The formative learning pipeline of future engineers  

Software engineering is not mere code fabrication.  
It is the disciplined practice of **design under constraints**.

This manifesto defines the values required to ensure that automation strengthens ‚Äî rather than weakens ‚Äî the sustainability of our profession (as software engineers).

# üìú Our Hierarchy of Values

We recognize the value of the items on the right.  
But we value the items on the left more.

| We Value | Over |
|-----------|------|
| **Human agency and accountability** | Opaque efficiency without traceability |
| **Explicit design decisions** | Functionally correct auto-generated implementation |
| **Critical evaluation and deep validation** | Practical plausibility |
| **Evidence-based rigorous validation** | Delivery speed |
| **Learning through cognitive friction** | Delegation convenience |
| **Team social capital (mentoring & collaboration)** | Brute-force computational power |
| **Structural prudence (Zero Trust)** | Tool/agent autonomy |

# üõ† What This Means in Practice

Endorsing this manifesto implies concrete commitments:

- AI-generated code remains subject to accountable human review.
- Design intent and trade-offs must be explicit before automation is delegated.
- Early-career engineers must develop design and validation competence.
- Organizations must preserve formative learning pathways.
- Critical systems require transparent validation criteria.
- Automation must elevate engineering judgment ‚Äî not displace it.

---

# ‚öñ Our Position

We are **not anti-AI**.

We are opposed to:

- Ungoverned automation  
- Opaque development pipelines  
- The erosion of engineering judgment in favor of productivity gains through IA 

AI and software engineering discipline are complementary and synergic.

But without deliberate governance, short-term efficiency can undermine long-term resilience.

The sustainability of software engineering requires:

- Human-in-the-loop design  
- Explicit accountability  
- Deep validation  
- Intergenerational knowledge transfer  

Software must remain intelligible, debuggable, and accountable.
